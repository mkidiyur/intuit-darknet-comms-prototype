CAS — Customer Self-Reporting & Self-Remediation Flow · Intuit Security Center

Feature Overview

The CAS (Customer Account Security) Self-Reporting & Self-Remediation flow empowers Intuit customers to independently report suspicious account activity and account takeovers (ATOs) without needing to call customer support. The feature integrates an AI-powered Intuit Assist chatbot into Security Center to guide users through a structured questionnaire, automated risk assessment, guided remediation steps, and escalation to a human investigation team when needed.

Business motivation: Support calls for suspicious activity reports are restricted to M–F 6am–6pm PST, leaving users exposed outside of those hours. This self-service flow provides 24/7 coverage, reduces support volume, and improves customer trust through transparency and immediate action.

User Flow

  1. User lands on Security Center — sees Security Score (45 Critical), suspicious activity banner, and incomplete security checklist
  2. User clicks "Report Suspicious Activity" button — Intuit Assist chatbot panel opens (Screen 2)
  3. Bot greets user empathetically and presents multi-select questionnaire about suspicious activity types
  4. User selects applicable items and clicks Continue
  5. Bot analyzes responses and presents risk level (HIGH RISK in this flow) with two action paths (Screen 3)
  6. Path A (Self-Remediation): User selects "Secure my account now" → Screen 4: Guided 3-step checklist (Password reset → MFA setup → Activity review)
  7. Path B (Escalation): User selects "Report to investigation team" → Screen 5: Form collects incident details → Case #IB-XXXX opened → Success confirmation
  8. Both paths converge to Screen 6: Post-remediation Security Center with updated score, completed checklist items, and remaining action prompt

Interaction Notes

  • Chatbot entry: Panel slides in from right using slideInRight animation (0.3s ease). Background content dims to rgba(0,0,0,0.1) to focus attention.
  • Questionnaire checkboxes: Toggle .checked class on label click. Label border turns blue, background turns #e8f4fb, check box fills blue with checkmark. "Continue" button is disabled until at least one option is selected.
  • Action path cards (Screen 3): Hover state adds blue border, blue box-shadow, slight translateY(-1px) lift. Hovered state shown statically in prototype to indicate default selection.
  • Risk level display (Screen 3): Computed server-side based on questionnaire responses. Score thresholds: 2+ high-impact options = HIGH, 1 high-impact = MEDIUM, informational only = LOW.
  • Progress bar (Screen 4): Animates width on step completion. Each step completion triggers score recalculation and updates the sidebar Security Score indicator.
  • Password strength indicator: Real-time evaluation of input value. 4-bar visual: 1 bar = Weak (red), 2 bars = Fair (yellow), 3 bars = Strong (green), 4 bars = Very Strong (green).
  • Security Score ring: SVG circle with stroke-dasharray/dashoffset calculation. Formula: dashoffset = circumference × (1 − score/100). Circumference = 2π × r = 238.76 for r=38.
  • Post-remediation score animation: On Screen 6 load, animate score ring from previous value (45) to new value (78) over 1.2s with easing.
  • Banners: × button hides banner with fade + slideUp animation. Dismissed state persists in localStorage for the session.
  • Case number format: #IB-[YYYY]-[4-digit-id]. Monospace font. Auto-copied to clipboard on click with tooltip confirmation.
  • Connected apps unverified state: Unverified apps shown with warning icon and yellow background. "Review" button triggers modal with app details and remove option.
  • Accessibility: All interactive elements have focus-visible outlines. Color is not the sole indicator of status (icons/labels accompany all color-coded states). ARIA labels on icon-only buttons.
  • Responsive breakpoints: Below 900px, sidebar collapses to icons only. Below 700px, main layout stacks vertically. Chat panel becomes full-width bottom sheet on mobile.
